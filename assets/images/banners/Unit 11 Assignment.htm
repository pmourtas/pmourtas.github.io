CIFAR-10 model assessment

 

1) Outline the method you used to partition the validation set from the training data and provide insights into the data specifics.

The validation set was created by splitting the CIFAR-10 (Al. Krizhevsky, 2009) training dataset into training and validation subsets using the train_test_split function from the sklearn.model_selection module (Scikit-learn, 2024). We set the test size to 0.1, so 10% of the original training data became a validation set. We also implemented the stratify parameter to ensure that the split will maintain the same proportion of data for each class in both the training and validation datasets. In that way, we will have more reliable performance metrics during model evaluation. Lastly, we set a parameter to ensure the reproducibility of our results (Deep AI, 2024).

Figure 1: Splitting the data (Part of the code, which is in the appendix)

 

As for the metadata, the CIFAR-10 dataset consists of 10 categorical classes, which are the airplane, automobile, bird, cat, deer, dog, frog, horse, ship, and truck. This dataset contains a total of 60,000 images with dimensions of 32 wide x 32 high pixels and 3 color channels: red, green, and blue. Each class contains 6,000 images. After splitting the datasets, the training set now contains 45,000 images, the validation set 5,000 images, and the test set 10,000 images.

Figure 2: Picture of the classes in the CIFAR-10 dataset (Al. Krizhevsky, 2009).

 

As for the results, our model shows a gradual improvement in both training and validation accuracy over the course of 86 epochs (UNext, 2024). The training accuracy started at 27.8% and improved to 73.4% by the final epoch, indicating that the model is learning from the training data. The validation accuracy demonstrated a similar upward trend, starting at 41.9% and reaching out to 78.9% by the last epoch. That means that our model is not only memorizing the training data but is also generalizing well to unseen data. That is, it learns to identify underlying patterns from new raw data (Fr. von Lehn, 2023).

Figure 3: First and last epochs result.

 

Figure 4: Model Accuracy and Loss over the epochs.

 

The final accuracy on our validation and test set is approximately 79%, which is relatively good for such a complex task like image classification. Our precision, recall, and F1 score are also around 79%, indicating a balanced performance. Precision indicates that when the model predicts a class, it is correct about 79.4% of the time. Recall indicates that it identifies about 78.9% of the actual instances of the classes, while the F1 score, the harmonic mean of precision and recall, is approximately 78.4%, indicating a solid overall performance (Google for Developers, 2024).

Figure 5: Precision, recall, and F1-score metrics.

 

Other than that, the classification report shows how our model performed on each class. Classes such as 1, 8, and 0 exhibit high precision and recall, indicating strong performance. However, Class 3 shows lower performance with precision 0.77 and recall 0.50, indicating that our model struggles to correctly identify instances of this class (Scikit-learn, 2024).

Figure 6: Classification Report

 

On the other side, the confusion matrix was also used to evaluate the performance of our classification model, but in a different way (Geeks for Geeks, 2024). In that matrix, the diagonal values represent the correct predictions where a true label matches the predicted label. For example, in the first row, the model correctly classified 823 samples as 0, while in the second row, 908 samples as 1. All the other values that are not diagonal represent misclassifications, or how many times the model incorrectly predicted a label. For example, in the first row, 50 instances where the true label was 0 were classified as 9. Our matrix indicates that the model is performing well because of the high diagonal values. Also, it tells us that class 9 has very high correct predictions with very few misclassifications, unlike class 3, which has more errors.

Figure 7: Confusion Matrix

 

Lastly, we also implemented a ROC (Receiver Operating Characteristic) curve, which also evaluates the performance of a classification model, but this time by analyzing the trade-off between the True Positive Rate (TPR) and the False Positive Rate (FPR) across different classification thresholds (Google for Developers, 2024). TPR is the recall, which measures the proportion of correctly predicted positive cases out of all actual positive cases, while the FPR measures the proportion of actual negatives incorrectly classified as positive. The diagonal line represents a random classifier, while the ROC curve, the pink line, depicts the models TPR against FPR for all possible thresholds. In our case, the Area Under the Curve (AUC) is 0.98, which indicates how well our model separates classes. Have in mind that a perfect classifier has an AUC of 1.00.

Figure 8: ROC Curve

 

Overall, our model demonstrates promising results with potential for enhancement through further training, optimization strategies, or even implementing more complex architectures (N. Mohsen, 2023).

 

2) Explain the importance of maintaining a separate validation set.

A separate validation set is crucial for evaluating our model's performance during training. By assessing the model's performance on unseen data through the validation set, we can detect if the model is overfitting the training data. That is, it will perform very well on the training set but poorly on unseen data. Additionally, it allows us to tune our models parameters without affecting the test set, ensuring a fair assessment of the model's generalization ability. Furthermore, with the implementation of early stopping, we can use the validation set to determine when to halt training in order to prevent overfitting. These practices are fundamental for developing robust and effective machine learning models (Geeks for Geeks, 2021).

3) Elaborate on the architecture of the artificial neural network.

Artificial Neural Networks (ANNs) are computational models that try to imitate the biological neural networks and the way that they work. ANNs consist of layers such as interconnected neurons, like input, hidden, and output layers. Each neuron is fully connected to every neuron, making lines with their interconnected layers. While ANNs can be used for simple classification problems and general-purpose tasks such as predicting numerical outcomes, they are not optimized for capturing spatial relationships within data. This limitation makes them less effective with high-dimensional data like images. ANNs treat their inputs as one-dimensional vectors, and because of that they do not maintain the spatial structure of their data. In that way, they are making the whole procedure more difficult for the network to learn and find relevant features. Additionally, they require manual feature extraction to identify important characteristics of the data, in contrast to Convolutional Neural Networks (CNNs) that learn the features automatically (Javapoint, 2024). For these reasons, we preferred to apply a CNN model rather than an ANN (K. Zoumana, 2024).

So, as it concerns our elaboration, we will proceed with our neural network architecture, which is a Convolutional Neural Network (CNN) designed specifically for image classification (SuperAnnotate, 2023). Our model consists of three convolutional layers with increasing filter sizes (32, 64, 128) to capture different and more complex features. Each convolutional layer is followed by MaxPooling and batch normalization. MaxPooling reduces the spatial dimensions of height and width of the input data while it retains the most important information (K. Dhanush, 2023). On the other side, batch normalization improves training speed, stability, and performance as it normalizes the inputs of each layer so that they have a mean of zero and a variance of one (S. Shipra, 2024). Additionally, our model includes a flattening layer that converts the 2D matrices from the convolutional layers into a 1D vector, and it is used at the end of our network in order to feed our penultimate dense layer (Geeks for Geeks, 2024). Our dense layer is a fully connected layer with 128 neurons that have their own sets of weight and bias, and it is followed by an output layer consisting of 10 neurons, one for each class, utilizing the softmax activation function that we implemented in order to convert the raw outputs into probabilities (A. Thakur, 2024). Softmax is specifically designed to handle multi-class classification scenarios like in our case (B. C. Priya, 2023).

 

Figure 9: MaxPooling, Relu, Softmax, BatchNormalization, Convolution layers, Flatten and Dense layers.

 

4) Delve into the chosen activation function.

The activation function used in the hidden layers is ReLU (Rectified Linear Unit). This function is preferred for its simplicity and efficiency that helps our model to learn complex patterns. ReLU introduces non-linearity into the model, which is crucial for learning complex patterns in data (Br. Whitfield, 2024). It provides faster training compared to other more complex activation functions. Additionally, as mentioned earlier, the final layer employs the softmax activation function, which is suitable for multi-class classification as it converts the output into probabilities across the classes.

 

5) Describe the loss function implemented.

The loss function we implemented is Sparse Categorical Crossentropy. This loss function is ideal for multi-class classification problems where the classes are mutually exclusive, meaning that each sample belongs to one and only one class, such as in our case. It is also suitable for situations where the target labels are provided as integer class labels instead of one-hot encoded vectors. Each image in our model's dataset has a corresponding label, an integer from 0 to 9, making Sparse Categorical Crossentropy a natural fit for our model. The function calculates the cross-entropy loss, discrepancies, between the true class labels and the predicted probabilities produced by the model during training. Overall, it helps simplify the data preprocessing steps, especially when dealing with large datasets (F. A. Moreno, 2021).

Figure 10: Loss function

 

6) Detail the number of epochs utilized in the modeling process.

Our model was trained for a maximum of 500 epochs. However, with the implementation of early stopping in conjunction with patience, our models training halted at the 86th epoch because the validation loss did not improve for 20 consecutive epochs. This approach helped us prevent overfitting while allowing our model to train for as long as necessary (J. C. Olamendy, 2024).

 

7) Share insights on your strategy regarding the neural network's design elements.

Our strategy was to use multiple convolutional layers to extract features while also employing dropout and L2 regularization to mitigate overfitting. Convolutional layers help capture spatial features in images through filters that can detect edges, textures, and more complex patterns as we progress deeper into the network. Dropout randomly sets some of the neurons to zero during training, ensuring that the model does not rely too heavily on specific neurons (H. Yadav, 2022). Meanwhile, L2 regularization penalizes large weights and encourages the model to learn simpler patterns (A. Nagpal, 2017).

Figure 11: Dropout & L2 regulazation

 

In addition, we added batch normalization after each convolutional layer to improve training speed and stability by normalizing the inputs, specifically the pixel values of the images. We also implemented data augmentation, utilizing ImageDataGenerator to artificially expand our training dataset (An. Bhandari, 2024). This introduced variations in the images, such as rotation, shifting, zooming, and flipping, making our model more robust. Additionally, by applying early stopping and model checkpoints, we were able to save the best training model and prevent unnecessary computations (Deepchecks, 2024).

Figure 12: ImageDataGenerator

 

Lastly, classification reports and confusion matrices were essential tools for evaluating our model's performance. Metrics such as precision, recall, F1 scores, and multi-class ROC curves provided insights into how well our model performs across different classes, allowing us to gain a deeper understanding of its capabilities (Deepchecks, 2024).

In conclusion, all these features contributed to building a robust model capable of generalizing well to unseen data. Meanwhile, the evaluation metrics provided a comprehensive view of our models performance, enabling us to make informed decisions regarding further adjustments and improvements.
